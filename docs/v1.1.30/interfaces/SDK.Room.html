Interface Room

The WebRTC Room is a virtual meeting place for Participants, who can send video/audio streams (ie, Tracks) to each other in real time.

Note: The Room model includes several "Hooks". For more detailed information, including examples, on how to use these hooks, please see the entry for useParticipants.

useParticipants

  • useParticipants(cb: ((participants: Participant[]) => void)): Disposable
    • Defined in src/core/types.ts:480

    Watches for changes on the Participants in a room, and runs a callback whenever the Participants change. Examples of events on which this callback would fire include (but are not limited to) a Participant joining or leaving the room, a track being published or unpublished, etc.

    example

    Here is an example of how this function (and the other hooks in this section) might be used in a vue component

    Vue
     
    export default {
      data() {
        return {
          participants: [],
          dispose: () => {},
        }
      },
      beforeMount() {
        this.dispose = useParticipants((participants) => {
          this.participants = participants
        })
      }
      beforeUnmount() {
        // Calling this.dispose will remove the event listeners that were added by `useParticipants` in `beforeMount`
        this.dispose()
      }
    }
    example

    These hooks work well with react hooks. Here is an example of how one might use this hook (and the other hooks in this section) within in a react component.

    React
     
    import React, { useEffect, useState } from 'react'

    const ParticipantList = () => {
      const [participants, setParticipants] = useState([])
      useEffect(() => {
        const dispose = useParticipants(setParticipants)
        // It is important to return dispose within useEffect in order to clean up our event listeners
        return dispose
      })
      return (
        <div>
          {participants.map((participant) => (
            <div>
              {participant.displayName}
            </div>
          ))}
        </div>
      )
    }
    example

    We also could have written the useEffect block more concisely as such:

    useEffect(() => useParticipants(setParticipants))

    Parameters

    • cb: ((participants: Participant[]) => void)

      Callback which will run whenever the Room's participants are updated. This callback essentially serves as an event listener.

        • (participants: Participant[]): void

        • Parameters

          • participants: Participant[]

            Array of all Participants in the Room

          Returns void

    Returns Disposable

    Function which, when invoked, stops watching Participants. Essentially a removeEventListener.

sendChatMessage

  • sendChatMessage(data: { message: string; metadata?: string | object; recipients?: string[] }): Promise<void>
    • Defined in src/core/types.ts:493

    Sends chat message to entire Room, or a private message (if specified) from local participant.

    To display these messages, see: {@link Room.useChatMessage useChatMessage}

    Parameters

    • data: { message: string; metadata?: string | object; recipients?: string[] }
      • message: string

        Content of the chat message

      • Optional metadata?: string | object

        Optional. Arbitrary data that can be sent as part of the message.

      • Optional recipients?: string[]

        Optional. Identities of the recipient participants. If left undefined, the message will be sent to all participants in the Room. (default: undefined).

    Returns Promise<void>

sendData

  • sendData(data: any, recipientIds?: string[]): Promise<void>
    • Defined in src/core/types.ts:513

    Send arbitrary data from the local Participant to other participants in the Room. This method can be used in conjunction with the onData method to build a custom system of events for your app.

    Parameters

    • data: any

      The data that is being sent to other Participants. This should be in the form of a "stringifiable" object/array, as JSON.stringify will be used on this parameter before it is sent to the other participants.

    • Optional recipientIds: string[]

      Optional. Array of IDs belonging to Participants who should receive the data. If left undefined, message will be sent to all Participants in the Room. (default: undefined)

    Returns Promise<void>
