Namespace SDK

Repository of interfaces and types provided by the Studio SDK.

Core models

Studio

Represents a singleton of the initialized Studio SDK, the basis of all core functionality.

import { init, Helpers } from `@api.stream/studio-kit`

// Initialize the studio
const studio = await init()

Project

Represents a single project owned by a user. A project provides access to a unique WebRTC room for video conferencing, and contains information about all elements of a stream configuration.

// Authenticate a user and fetch their existing projects
const { projects } = await studio.load(accessToken)

// Create a new project
const project = await Helpers.ScenelessProject.create()

Related: ScenelessProject

Room

Represents a WebRTC video conferencing room that allows multiple participants to join and share video feeds using webcam and screenshare.

// Join the WebRTC room associated with a project
const room = await project.joinRoom()

User

User: { id: string; projects: Project[]; props: Metadata }
  • Defined in src/core/types.ts:531

The Studio SDK's User model is rather minimal, as Users should be managed by your own service.

Type declaration

  • id: string

    Unique alpha-numeric ID. Example: "1234a56b7890cd12e34f5gh6". Corresponds to Live API collection_id.

  • projects: Project[]

    Array of Projects belonging to the User.

  • props: Metadata

    Arbitrary data stored on the user. Opaque to the SDK.
