Interface Studio

The root of the initialized SDK.

Created by a call to init().

Typically, this should be followed by a call to Studio.load().

import SDK from `@api.stream/studio-kit`

// Get a Studio instance
const studio = await SDK.init()

// Invoke studio.load() to load the current user's projects
const { projects } = await SDK.load(accessToken)

Note: If the user is acccessing a project using a guest access token, loading the user is not neccessary. The associated project will be accessible via Studio.initialProject.

render

render: (settings: CompositorSettings) => Promise<void>

Type declaration

    • (settings: CompositorSettings): Promise<void>

    • Render the output compositor displaying the stream canvas, which will be used to display the live feed once a user starts their broadcast. Renders into a supplied HTML element.

      This compositor may double as an interactive editor with optional settings.

      Note: The compositor will automatically render at the largest possible size accomodated by the element that is passed as its container. If the container is smaller than the project resolution (e.g. 720px x 1280px), all of the canvas elements will scale down automatically to fit.

      Parameters

      • settings: CompositorSettings

Type declaration

    • (cb: <Key>(name: Key, payload: any) => void): Disposable

    • Subscribe to all Events emitted by the intialized Studio.

      View the EventMap to see the payload associated with each event.

      studio.subscribe((event, payload) => {
        if (event === 'BroadcastStarted') {
          setIsLive(true)
        } else if (event === 'BroadcastStopped') {
          setIsLive(false)
        } else if (event === 'BroadcastError') {
          setErrorMessage(payload.error)
        }
      })

createDemoToken

  • createDemoToken(options: { name: string; serviceName: string; userId: string }): Promise<string>

  • Create a token that is valid only for demo purposes.

    Note: In a production system, this token must be granted as part of the user login flow (wherein a partner front-end logs into a partner backend, which in turn connects to the Lightstream backend to obtain an appropriate access token, which is then returned by the partner backend to the partner front-end for SDK use).

    Pass into Studio.load to receive the assocaited demo User.
